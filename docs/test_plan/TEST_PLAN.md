# 测试计划文档 (Test Plan)

**项目名称**: wangdanatest.top 自动化测试项目
**目标网站**: https://www.wangdanatest.top
**版本**: 1.0.0
**编写日期**: 2026-02-21
**测试框架**: Playwright + Node.js

---

## 1. 测试目标

本项目旨在为 wangdanatest.top 网站建立全面的自动化测试框架，覆盖以下层面：

- **功能测试 (Functional Testing)**: 验证核心业务功能是否按预期工作
- **接口测试 (API Testing)**: 测试后端REST API的可靠性和数据完整性
- **安全测试 (Security Testing)**: 验证应用安全防护机制（XSS、CSRF等）
- **性能测试 (Performance Testing)**: 确保页面加载时间和响应速度符合要求
- **兼容性测试 (Compatibility Testing)**: 验证跨浏览器、跨设备的表现一致性
- **可访问性测试 (Accessibility Testing)**: 确保应用符合WCAG标准

---

## 2. 测试范围

### 2.1 被测试功能

| 功能模块 | 描述 | 测试类型 | 优先级 |
|---------|-----|--------|------|
| **主页** | 首页加载、菜单导航、基础布局 | 功能/兼容性 | P0 |
| **关于页面** | 关于页面导航和内容显示 | 功能 | P1 |
| **搜索功能** | 搜索框输入、结果展示、分页 | 功能 | P0 |
| **登录流程** | 用户认证、会话管理、错误处理 | 功能/安全 | P0 |
| **注册功能** | 用户注册、表单验证、邮箱确认 | 功能 | P1 |
| **论坛API** | 讨论列表、标签API、数据获取 | 接口 | P0 |
| **安全防护** | XSS防护、CSRF防护、认证授权 | 安全 | P0 |
| **性能指标** | 页面加载时间、响应延迟 | 性能 | P1 |
| **移动适配** | 响应式设计、移动设备运行 | 兼容性 | P1 |

### 2.2 测试环境

| 环境 | 浏览器 | 设备 | 操作系统 |
|-----|------|------|--------|
| **桌面** | Chrome, Firefox, Safari | 标准分辨率 | Windows/Mac/Linux |
| **移动** | Chrome Mobile, Safari Mobile | Pixel 5, iPhone 12 | Android/iOS |
| **CI/CD** | Chrome (Headless) | Docker Container | Linux |

## 4. 测试执行计划

### 4.1 执行周期

- **每日测试**: 所有 P0 优先级用例，提交代码时执行
- **周测试**: 所有功能和安全用例，每周执行一次
- **月测试**: 完整测试套件，包含性能和兼容性测试
- **发布测试**: 新版本发布前执行完整测试

### 4.2 执行环境

```
本地开发环境:
- Node.js >= 18.x
- npm >= 9.x
- Playwright >= 1.40.x
- 操作系统: macOS / Windows / Linux

CI/CD 环境:
- GitHub Actions
- Docker 容器化
- 自动运行所有测试
- 生成测试报告
```

### 4.3 测试命令

```bash
# 运行所有测试
npm test

# 运行特定类型的测试
npm run test:ui          # UI 模式（交互式）
npm run test:api         # API 测试
npm run test:security    # 安全测试
npm run test:performance # 性能测试

# 生成测试报告
npm run test:report      # HTML 报告

# 特定浏览器测试
npm run test:chromium    # 仅 Chrome
```

## 6. 缺陷管理

### 6.1 缺陷分类

| 级别 | 描述 | 处理方案 |
|-----|------|--------|
| **Critical** | 系统崩溃、核心功能不可用 | 立即修复，停止发布 |
| **High** | 重要功能失效、数据丢失 | 优先修复，延期发布 |
| **Medium** | 功能部分失效、性能降低 | 计划内修复 |
| **Low** | UI 瑕疵、非关键功能 | 后续版本修复 |

### 6.2 缺陷报告

所有缺陷通过 GitHub Issues 记录，格式如下：

```markdown
**标题**: [Test Failure] TC-HOME-001: 验证主页标题

**类型**: Bug
**优先级**: High
**环境**: Chrome 120, macOS 14.2

**测试用例**: TC-HOME-001
**错误信息**: Assertion failed: expected 'Wangdana' but got 'undefined'
**复现步骤**:
1. 访问首页
2. 检查页面标题
3. 标题不正确

**建议**: 检查 meta 标签配置
```

## 9. 风险和缓冲

### 9.1 已知风险

| 风险 | 影响 | 概率 | 缓冲方案 |
|-----|------|------|--------|
| 网站不稳定 | 测试失败 | 低 | 启用重试机制 |
| 测试数据丢失 | 无法登录 | 低 | 维护备用账户 |
| 浏览器更新 | 兼容性问题 | 中 | 定期更新依赖 |
| 网络延迟 | 超时失败 | 中 | 调整超时时间 |

### 9.2 缓冲时间

- 每个test用例添加 500ms ~ 2s 的合理等待时间
- 不稳定步骤启用重试 (retry: 2)
- 登录操作使用专用登录方法确保可靠性
